# Claude Rules Init

IMPORTANT: You MUST fetch rule files from GitHub. Do NOT generate rules from your own knowledge.

Rules source base URL: `https://raw.githubusercontent.com/lifedever/claude-rules/main/`

## Workflow

### Step 1: Detect Tech Stack

Scan the current project directory to identify languages and frameworks:

| Signal | Rule File |
|--------|-----------|
| `tsconfig.json` or `*.ts` files exist | `languages/typescript.md` |
| `package.json` contains `"vue"` in dependencies | `frameworks/vue.md` |
| `package.json` contains `"react"` in dependencies | `frameworks/react.md` |
| `src-tauri/` directory exists | `frameworks/tauri.md` |
| `*.js` files exist without `tsconfig.json` | `languages/javascript.md` |
| `*.java` files or `pom.xml` exists | `languages/java.md` |
| `build.gradle.kts` or `*.kt` files exist | `languages/kotlin.md` |
| `pom.xml`/`build.gradle` contains `spring-boot` | `frameworks/springboot.md` |
| `Package.swift` or `*.swift` files exist | `languages/swift.md` |
| Swift files contain `import SwiftUI` | `frameworks/swiftui.md` |
| `pyproject.toml` or `requirements.txt` exists | `languages/python.md` |
| `*.html` files exist | `languages/html.md` |
| `*.css`/`*.scss` files exist | `languages/css.md` |

Present detection results and ask user to confirm or adjust before proceeding.

### Step 2: Fetch Rules from GitHub

You MUST use WebFetch to download each rule file. Do NOT skip this step. Do NOT write rules from memory.

Construct URLs by appending the rule file path to the base URL. Example:
- `https://raw.githubusercontent.com/lifedever/claude-rules/main/base/core.md`
- `https://raw.githubusercontent.com/lifedever/claude-rules/main/languages/typescript.md`

Always fetch these two files:
1. `base/core.md`
2. `base/git.md`

Then fetch each detected language and framework file.

When calling WebFetch, use prompt: "Return the complete raw file content exactly as-is, do not summarize or modify."

If a fetch fails, warn the user and skip that file.

### Step 3: Assemble & Write CLAUDE.md

Concatenate ALL fetched content in this exact order:

1. First line: `<!-- Generated by claude-rules-init | https://github.com/lifedever/claude-rules -->`
2. Empty line
3. Content of `base/core.md`
4. Content of `base/git.md`
5. Content of each detected language file
6. Content of each detected framework file

Write the assembled content to `CLAUDE.md` in the project root.

If CLAUDE.md already exists, warn the user and ask before overwriting.

### Step 4: Report

Show:
- List of fetched rule files
- Total line count of generated CLAUDE.md
- Remind user they can re-run `init rules` anytime to pull the latest upstream updates
