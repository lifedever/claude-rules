# Claude Rules Init

Generate a project-specific CLAUDE.md by detecting the tech stack and fetching the latest rules from GitHub.

Rules source: `https://raw.githubusercontent.com/lifedever/claude-rules/main/`

## Workflow

### Step 1: Detect Tech Stack

Scan the current project directory to identify languages and frameworks:

| Signal | Rule File |
|--------|-----------|
| `tsconfig.json` or `*.ts` | languages/typescript |
| `package.json` with `"vue"` dep | frameworks/vue |
| `package.json` with `"react"` dep | frameworks/react |
| `src-tauri/` directory | frameworks/tauri |
| `*.js` without `tsconfig.json` | languages/javascript |
| `*.java` or `pom.xml` | languages/java |
| `build.gradle.kts` or `*.kt` | languages/kotlin |
| `pom.xml`/`build.gradle` with `spring-boot` | frameworks/springboot |
| `Package.swift` or `*.swift` | languages/swift |
| Swift files with `import SwiftUI` | frameworks/swiftui |
| `pyproject.toml` or `requirements.txt` | languages/python |
| `*.html` files | languages/html |
| `*.css`/`*.scss` (non-Tailwind) | languages/css |

Present detection results and ask user to confirm or adjust before proceeding.

### Step 2: Fetch Rules

Use WebFetch to retrieve each rule file from GitHub raw URL.

Always fetch:
- `base/core.md`
- `base/git.md`

Then fetch detected language and framework files (e.g. `languages/typescript.md`, `frameworks/vue.md`).

If a fetch fails, warn and skip.

### Step 3: Assemble & Write CLAUDE.md

Concatenate in order: header comment → base → languages → frameworks.

Header: `<!-- Generated by claude-rules-init | https://github.com/lifedever/claude-rules -->`

If CLAUDE.md already exists, warn the user and ask before overwriting. Back up to CLAUDE.md.bak if confirmed.

### Step 4: Report

Show included files, total lines, and remind user to re-run anytime to pull upstream updates.
