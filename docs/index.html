The Problem

AI mimics your worst habits

When AI reads your legacy code, it thinks that's the standard. It copies every anti-pattern it finds.

💣

God Classes keep growing

800-line file? AI adds 50 more lines instead of refactoring into smaller modules.

💥

any spreads like a virus

One any in your codebase, AI uses it everywhere. Type safety goes out the window.

🔨

Callback hell multiplies

5-level nesting? AI nests 6. It treats your worst code as the template.

⚠️

Outdated patterns persist

Using ObservableObject in Swift 6? Old enum in TypeScript? AI follows suit.

The Difference

Vague vs. actionable

Most coding rules are too vague for AI to execute. Ours are concrete directives with code examples.

❌ Other rule libraries

  • Use proper error handling
  • Implement appropriate caching
  • Use proper navigation methods
  • Keep components organized
  • Follow best practices

✅ claude-rules

  • Functions ≤30 lines, files ≤300 lines
  • ref for primitives, reactive for objects
  • No any — use unknown then narrow
  • No barrel exports (causes circular deps)
  • as const + union type, not enum

Get Started

Up and running in 60 seconds

1

Add marketplace & install plugin

# Add the marketplace
claude plugin marketplace add lifedever/claude-rules

# Install the plugin
claude plugin install init-claude-rules@claude-rules
2

Restart Claude Code

The plugin needs a restart to register the skill.

3

Run in any project

/init-rules
